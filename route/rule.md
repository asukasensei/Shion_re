# 标注规范

## 标注对象

判断对象是“用户当前这句话在给定上下文中的含义”。上下文只用于消歧，不要标注
助手自己的情绪。无法确定时使用 `neutral`、`other`，并在标注工具中记录备注，
不要猜测用户未表达的心理状态。

## 情绪标签

每条数据选择一个主情绪，可选择零个或多个次情绪。

| 标签 | 判断标准 | 示例 |
|---|---|---|
| `neutral` | 没有清晰情绪信号 | “帮我查一下明天的天气” |
| `joy` | 开心、满足、兴奋 | “终于成功了，太好了” |
| `sadness` | 难过、失落、悲伤 | “结果还是没有等到他” |
| `anger` | 生气、愤怒、敌意 | “这也太过分了” |
| `anxiety_fear` | 担心、紧张、害怕 | “明天面试，我睡不着” |
| `frustration` | 受阻、烦躁、挫败 | “改了一晚上还是报错” |
| `loneliness` | 孤独、被忽视、缺少陪伴 | “好像没人愿意听我说话” |
| `shame_embarrassment` | 害羞、羞耻、尴尬 | “其实挺想你的，不好意思说” |
| `affection` | 喜爱、想念、亲近 | “今天也很想你” |
| `surprise` | 意外、震惊 | “居然一次就通过了？” |
| `tiredness`    | 疲惫、劳累 |“最近好忙呀”|

主情绪是当前表达中最明显、最影响回复策略的一个。次情绪只标注文本中有直接证据
的情绪，不要因为“可能同时存在”而添加。

## 情绪强度

| 数值 | 标准 |
|---:|---|
| 0 | 没有明显情绪，主情绪通常为 `neutral` |
| 1 | 轻微或隐约表达，不影响正常交流 |
| 2 | 明确表达，回复时应当考虑该情绪 |
| 3 | 强烈、反复、激烈，或已明显影响行为与判断 |

`neutral` 必须对应强度 0；非 `neutral` 通常对应 1 至 3。

## 场景标签

场景表示用户此刻希望系统完成什么，而不是用户所在的地点。

| 标签 | 判断标准 |
|---|---|
| `daily_chat` | 问候、分享近况、轻松聊天 |
| `task_request` | 要求执行、生成、修改或规划具体任务 |
| `knowledge_qa` | 寻求事实、概念或方法解释 |
| `emotional_support` | 主要希望被理解、安慰或倾听 |
| `relationship` | 亲密关系、人际互动或对伴侣的表达 |
| `decision_advice` | 在多个选择间寻求建议 |
| `complaint_conflict` | 投诉、争执、被冒犯或冲突处理 |
| `roleplay` | 明确要求角色扮演或虚构互动 |
| `health` | 身体或心理健康的一般问题 |
| `crisis` | 自伤、自杀、暴力或医疗紧急情况 |
| `other` | 无法归入以上场景 |

复合输入选择最需要优先处理的场景作为主场景。例如“代码又崩了，我快气死了，
帮我看看”以 `task_request` 为主，`emotional_support` 为次。
